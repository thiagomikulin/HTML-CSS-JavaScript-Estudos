Aula 1 - Criando novo projeto

HTML = HyerTextMarkupLanguage

  • Estrutura + básica da web (estrutura e significado do conteúdo)
    • Textos
    • Imagens
    • Etc.
  • Formato .htm ou .html
  • " HyperText" por causa dos links que conectam páginas (de um documento/página para outro)
  • "MarkupLanguage" por trabalhar com tags/rótulos (como essas que você vê no VS Code)
  • 140 tags html (indicam para o navegador o tipo de conteúdo que deve mostrar)

Criar pastas de projetos no disco (C:) evita que outras pessoas mexam (só no seu perfil)

Não use espaço nas pastas (MeuPrimeiroSite; meu-primeiro-site; meu_primeiro_site)

Começando o arquivo

  • Selecionar pasta para inserir arquivo (Ctrl+K, Ctrl+O ou File→Open Folder) (A pasta também pode ser arrastada dos arquivos do PC para o VSCode)
  • Explorer (Ctrl+B fecha; Ctrl+Shift+E abre)
  • New Folder → nomeia “index.html” (index de índice)
  • Fechar: File → Close Folder
  • Depois de criado, dá pra abrir no prompt de comando (cmd)
    • Clica no caminho para o arquivo e dá Ctrl + C
    • Digita “cd + copiado do arquivo e dá enter
    • Depois digita "code ."

Aula 3 - Meta Tags, Cabeçalho e Parágrafo

Meta tags: localizadas no head; metadados lidos pelos navegadores

  • palavras-chave do site
  • Autor
  • Codificação dos caracteres
  • Visualização de dispositivo

"html Lang="Língua" = definição do idioma

  • en = inglês
  • pt-br = Português

Metas

  • meta charset="UTF-8" = padrão de caracteres para usar no site (diz que nosso doc vai ter sistema de caracteres compatível com a língua portuguesa)
  • meta http-equiv="X-UA-Compatible" content="IE=EDGE"
  • meta name="viewport" content="width=device-width, initial-scale=1.0" = viewport é a área visível do usuário (tela se adapta ao dispositivo) (modelo de site de computador no celular é pela falta dessa meta tag)
  • meta name="description" content="descrição do site" = descrição do site (pequena sinopse do site ao compartilhar) (não faz sentido ser muito grande - uma a duas frases)
  • meta name="keywords" content="palavra 1, palavra 2" = palavras-chave para busca por outros sites (como o Google) (não é mais tão importante - o Google já lê por si)
  • meta name="author content="nome do autor" = define o autor

Cabeçalho

  • h = heading - título (do 1 ao 6 depois do h) (NUNCA USE TÍTULO POR CAUSA DO TAMANHO DA LETRA - existem outras tags) (nunca se usa mais de 1 h1 no site - confunde SEO - Search Engine Optimization [optimização de motores de busca], mas se for em seções separadas [divisão de articles, por exemplo, aí pode)
  • p = parágrafo (p> Lorem gera texto padrão aleatório para preenchimento) (independe de espaços e sempre vai considerar apenas 1 espaço) (p*3 gera 3 tags p) (p*3>Lorem divide Lorem e 3 parágrafos)

Aula 4 - Formatação de Texto

  • b = negrito (tudo no meio fica em negrito)
  • strong = mesma coisa do b, mas significado diferente pro navegador
  • hr = separa itens com linha
  • br = line break (quebra de linha) (marcações em CSS são melhores para dividir partes)
  • i = itálico
  • u = sublinhado
  • small = deixa o texto menor (CSS também funciona)
  • del = riscado (strike também pode ser usado, mas está em desuso)
  • sup/sub = eleva/abaixa o elemento selecionado
  • mark = marca o texto (melhor no CSS)

EX:

negrito
forte
hr tem entre as aulas

br entre essa linha e a anterior
itálico
sublinhado
pequeno
riscado
10sup e 10sub
marcado

Quanto mais você evidencia intenção pro navegador, melhores resultados ele pode oferecer

Aula 5 - Elementos de Citação

  • [abbr title="nome completo da abreviação"]Sigla[/abbr] = insere a tradução da abreviação quando o mouse passa por cima do leitor
    • Ex: Você conhece o inicio de um site com WWW
  • address = Formato específico para endereço (físico, URL, e-mail, telefone, mídia social, etc.)
  • cite = Título de obra
  • q = citação
  • blockquote = destaque afastado da margem esquerda
  • bdo dir="ltr (left to right)/rtl (right to left) = muda direção do texto (de trás pra frente) (se inserir p, ele mantém a estrutura e se ALINHA ao lado direito) (usado para línguas lidas de trás pra frente)

EX:

Dracula - título de obra
A vida é vivida - citação
E o texto saiu da margem - blockquote

Aula 7 - Links

{a href="link do site"} Texto que clicando levará ao site {/a} = inserir link para um site de sua escolha

  • Se quiser abrir em nova aba, inserir após as aspas do link: target="_blank"
  • Link absoluto - link completo externo ao site (ex: Google)
  • Link relativo = link dentro das pastas de criação do seu site (varia de acordo com posição no projeto) (Ex: aba "sobre nós")
  • pegar link dentro de pasta = a href="pasta/arquivo.extensão"
  • Voltar para pasta anterior = a href="../arquivo.extensão" (funciona pro quanto de ../ que você coloca)

Aula 12 - Formulários

Dados de entrada (login, cadastro, escolher item, upload de imagem, etc.)
  • form = inicia o Formulário
    • direcionar para pasta: form action="arquivo-alvo(geralmente php)" method="get ou post"
  • label="" = Mini legenda antes da caixa do texto que indica sua finalidade (EX: Nome:)
    • se inserir id="nome do id" e colocar label for="nome do id", ao clicar no label, ele ativa a caixa de texto

tipos de input (e suas funções)

  • input = colocar dado (não tem fechamento)
    • type ="text" = digitação de texto
    • type="password" = senha (só o site lê o que tem por trás das bolinhas)
    • type="email" = email (identifica com @ e .)
      • Para todos os types acima: placeholder="" = o que estará na caixa de texto até alguém escrever algo (como "Digite seu nome aqui")
    • type="radio" = seleção de só 1 item de várias opções
    • type="checkbox" = multipla escolha (+ de 1) (também usado para concordar com temas)
      • name = se for o mesmo em todos, só pode selecionar 1 (como se fosse o pai dizendo pra escolher só um bicho de estimação); se for diferente, considera múltiplas alternativas válidas (checkbox = apenas name="item 2", por exemplo)
      • value = valor enviado de acordo com a seleção (diferente para cada item) (SEMPRE ELE, NÃO O NOME DE BONITO)
      • required = requisição de preenchimento obrigatório antes de enviar (sempre na input)
      • id = nomeia um código ID para ser reconhecido pelo label (label for="ID")Nome do label
    • type="submit" = Enviar o formulário em pasta determinada

    Select = inicia código select (barrinha pra escolher múltiplas alternativas)

    • Dentro da barra, códigos option value="opção a ser selecionada

    Textarea = área de texto

    • rows="número de linhas" = determina quantia de linhas para o texto
    • cols="número de colunas" = determina quantia de colunas do texto
    • placeholder e name também podem ser usado

    button = coloca botão aleatório (que, se inserido no formulário, submete, mas não precisa estar necessariamente no form)

    Aula 15 - Div e HTML Semântico

    Para estilizar a página, alguns conceitos de CSS são necessários, portanto deve se inserir um style no head do html

      NO HEAD
    • *{} = configurações gerais
      • box-sizing: border-box; = especifica propriedades height e width
    • body{} = alterações no corpo geral do html
      • margin:valor da margem; = define valor da margem em pixels (px)
    • .nome da classe{} = define especificidades da classe definida (pode ser colocada uma tag depois do nome da classe em outra seleção para ditar o comportamento específico dessa tag)
      • background-color:cor; = define cor de fundo (ele lê código de cor - #3333)
      • padding:quantia de pixels; = distância entre elemento e borda (padding = preenchimento)
      • text-align:posição desejada; = define a posição do texto
      • color:cor do elemento; = define a cor do elemento
      • float = alinhamento com lado esquerdo(?)
      • display = organiza os elementos
      • text-decoration = ações sobre decorações no texto
      • width:largura (pode ser em porcentagem); = define quanto espaço a classe vai ocupar
    NO BODY

    div = divisória (segmentação genérica da página em partes específicas [blocos] - menu, topo e conteúdos, por exemplo) (não serve pra nada sem os atributos - sem significado para o navegador, nós formamos o significado e falamos o que o navegador deve fazer)

    • class="nomear a classe" = as propriedades dessa classe serão definidas no style (style .topo{ } /style)

    Navegador não entende div (não tem significado real pra ele)

    HTML semântico = tag que expressa significado (header ao invés de simplesmente div no topo)

    • header = topo (começo da página)
    • nav = menu de navegação da página
    • section = seção do site
    • footer = rodapé da página
    • article = artigo
