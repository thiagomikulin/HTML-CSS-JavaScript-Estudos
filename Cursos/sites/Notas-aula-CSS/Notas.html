Aula 1 - O que é CSS e CSS Inline

CSS define estilo, cores, fontes, parte estética da Web

CSS = Cascading Style Sheets (folha de estilo em cascata)


Casamento perfeito com html

O CSS é responsável pela parte estética (cor de fundo, cor de texto, tamanho da fonte, espaçamentos, etc.)

Desenvolvido pelo W3C (World Wide Web Consortium) em 1996, pois o HTML não tinha tags para deixar a página mais visualmente bonita

CSS como complemento para o HTML (apesar de ser OUTRA linguagem de marcação)

Arquivos .css

CSS Inline

  • Style dentro de tag específica (como body, h1, etc.) = afeta APENAS elementos dentro dessa tag
    • "background:cor" = define cor de fundo
    • font-size:valor em px; = modifica tamanho da fonte (GERAL, TUDO AUMENTA OU DIMINUI)

Aula 3 - CSS Externo

Geralmente, é imprático colocar várias config de CSS em cada arquivo, portanto tem um jeito mais fácil por CSS Externo (um arquivo CSS na página que pode ser vinculado ao arquivo para aplicar suas propriedades)

  1. Criar uma pasta "nome.css"
  2. colocar as configs de style desejadas (direto, não precisa colocar style)
  3. criar um novo html
  4. nesse novo html, no head, colocar link rel="stylesheet" href="pasta de origem do css"
    • rel = relação (estabelece relação entre arquivos, nesse caso, um html e um css)
    • rel="stylesheet" = determina que o arquivo escolhido será responsável pelo estilo da página

É interessante criar várias folhas de estilo para cada função (uma pra certos títulos, outra pra cores, outra pra fontes, etc)

Hierarquia de prioridade do CSS: Inline > Interno > Externo

Pra não bagunçar, use sempre um padrão (recomendado o externo)

Aula 4 - Seletores e Classes

seletor = dizer pro CSS qual elemento você quer trabalhar (especificar p = modifica todos os p) (seletores no CSS podem ser conjuntos EX: h1,p{})

Classe = determina um padrão a ser reconhecido pelo CSS (se determinar "x" e colocar propriedades específicas para x, apenas x será modificado) (NO CSS: .nome da classe{propriedades}) (Elementos podem ter mais de uma classe) (nome nunca pode começar com número)

Para itens específicos = id (nome único da tag) (no CSS: #nome do id) (nome nunca pode começar com número)

Seletor universal: *{}

Aula 6 - Propriedade Background

Serve tanto para cor sólida quanto para imagem

  • Background color = cor de fundo (elemento geral - body - ou específico - h1)
  • Background image = Colocar imagem de fundo da seleção = background-image:url(localização da imagem);
    • se a imagem for muito grande: background-size: contain (reduz ao tamanho, mas repete padrão se faltar de algum lado) ou cover/100% (tenta adaptar ao tamanho requisitado)
      • background-repeat:no-repeat; = bloqueia a repetição, mas se for contain, deixa o espaço incompleto
      • background-repeat:repeat-x; (repete no eixo x)OU repeat y; (repete no eixo y)
      • background-position:posição da imagem = determina uma posição para a imagem

Se quiser facilitar, dá pra botar tudo isso em uma única tag background no CSS (EX: background: cor url() no-repeat center) (só o size seria externo a essa tag)

Aula 7 - Propriedade Border

Border = contorno de um elemento (borda)

  • tamanho em px
  • dá pra definir a cor
  • Tipos de borda
    • solid = barra reta
    • dotted = pontilhada
    • dashed = com traços
    • double = borda dupla
    • Groove = efeito 3D pra dentro
    • ridge = com 3D pra fora
    • inset = afundado de cima
    • outset = afundado de baixo
    • none = nada
    • hidden = escondido
  • Dá pra mesclar bordar (definindo aspectos para borders top, bottom, left e right)
  • para arredondar bordas = border-radius:quantia em px ou % (diferentes, mas ambos funcionam - px é mais recomendado)
  • não funciona em hr

Aula 14 - Propriedade Display

Controle de layout (se ou como um elemento será exibido) (já tem um valor padrão de acordo com o tipo de elemento)

Tipos de Display

  • Bloco = começa no início da linha e ocupa todo o espaço da página - padrão em bloco:
    • div
    • h1 até h6
    • p
    • form
    • header
    • footer
    • section
    • ul/ol e li
  • Inline = elemento não inicia em nova linha nem ocupa toda a extensão da página, mas se estende na linha conforme a largura necessária - padrão em inline:
    • a href
    • span = conteiner genérico de conteúdo fraseado
    • img
  • None = elemento não aparece na página (como os comentários na linha de código) (usado em elementos dinâmicos para desaparecer/aparecer objetos, de acordo com a interação)
  • Inline-block = mistura a união do inline com os parâmetros de height e width do block

Para mudar o display = display:inline/block/etc (quando o display é alterado para inline e tem blocos no meio do caminho do código [enter, por exemplo], esse elemento é convertido em espaço [resolve se colocar os códigos um do lado do outro - COPIA O CÓDIGO EXATO])

Inline não reconhece height nem width nem margin (ACEITA PADDING, POR EXEMPLO)(depende do conteúdo do elemento em si) (só display em block entende)

Aula 16 - Propriedade Position

Position = posicionamento do elemento em relação a tela

5 tipos de position: static, relative, fixed, absolute, sticky

  • static = posicionamento parado (padrão)
  • relative = posicionamento relativo àonde está o elemento
    • depois de colocar relative, pode ser inserido um deslocamento para o elemento escolhido (EX: left:20px; = se desloca 20px para a direita, se afastando da esquerda)
    • Nesse caso, do contrário do margin, os elementos se sobrepoem ao invés de serem deslocados juntos
    • z-index = usado para indicar quem está na frente e quem está atrás (quando sobrepostos) (ordem numérica - menores ficam atrás) (CAMADAS DE ELEMENTOS)
  • fixed = elemento fixo na tela, independente da rolagem (com coordenadas relativas à tela [topo, base, direita e esquerda])
  • absolute = posição do elemento em relação a elemento-pai (divs dentro do container, no exemplo)
    • Para funcionar, o elemento-pai precisa ter qualquer posicionamento MENOS static (se não, ele usa o body como referência)
  • sticky = gruda na tela e acompanha a rolagem (depois do ponto inicial)
    • se estiver em elemento-pai, segue até o limite deste

O POSITION SEMPRE SEGUE COMO BASE DE DESLOCAMENTO A POSIÇÃO ORIGINAL

Usado para efeitos em sites (Cards, por exemplo)

Aula 18 - Propriedade Float

Faz com que um elemento flutue/saia do fluxo da página para outro lugar(para alinhar, por exemplo)

Pode ser esquerda ou direita

Elemento flutuante se desprende da div

  • Para que a div absorva: overflow auto na .div (precisa pegar extensão total mesmo que o elemento esteja flutuando)

Clear = desconsidera a avaliação para seguir o fluxo (pula o espaço do lado do flutuante e continua depois)

  • both = coloca a borda de cima do elemento estilizado na borda de baixo do elemento flutuante

Float também pode ser usado para botões isolados - menu com itens à esquerda e à direita (dá pra mudar as propriedades individualmente também)

SE MARCAR FLOAT LEFT EM ITEM QUE VEM DEPOIS, ELE VIRA O 1º (por estar flutuando "FORA" do texto)

Se todos os elementos de um div flutuarem, ele não tem altura e "dá uma bugada"

Aula 21 - Especificidade CSS

Entre duas tarefas conflitantes em um mesmo elemento, qual o navegador vai executar?

Em ordem:

  1. Comandos diretamente na tag
  2. Estilo da página (no head, por exemplo)
  3. Estilo.css na página externa

Em ordem de seletores

  1. ID
  2. Classes
  3. Seletores simples (h1, p, li, etc.)

A ordem também faz efeito (se colocar na ordem do elemento - EX: h1 .red - ele ganha de uma classe sozinha, mas se for na ordem inversa ele perde [por falta de especificidade])

ORDEM DEFINITIVA DE FORÇA

  1. Style inline
  2. ID
  3. Seletor simples + classe
  4. Classe
  5. seletor simples

Se for vários comandos com seletores iguais e comandos diferentes, o último sempre terá prioridade

TIPOS DE GRADIENTE

  • Gradiente linear
    • Cores seguem a ordem de colocação

    • Direção = to direção (left, right, top, bottom [padrão])

Aula 27 - Flexbox

Conjunto de regras para deixar o site responsivo

Propriedade display tipo Flex

FRAMEWORK é mais fácil p/ montar site (só HTML/CSS = montar na mão)

Se adapta de forma automática

  • Display:Flex = um item do lado do outro
    • inline-block e float são antigos em comparação a este
  • Flex-direction = aponta a direção dos itens (em columns: comportamento em bloco; em rows: comportamento "inline float")
    • Pode ser combinado com media queries (em tamanhos menores em columns, em maiores em rows)
    • Dá pra indicar também se as rows ou colums são reversas ou não
  • Flex-wrap = envolve necessariamente todos os itens (mesmo que não apareçam na tela) no container(padrão nowrap)
  • Flex-flow = envolve text-wrap e text-direction
  • Justify-content (padrão = flex-start)
    • Flex-start = alinha na esquerda (começo) da página
    • Flex-end = alinha na direita (final) da página
    • Center = alinha no centro da página
    • Space-between = espaçamento uniforme entre os elementos
    • Space-around = espaçamento uniforme de toda a volta dos elementos
  • Align-items = posicionamento vertical dos itens INDIVIDUAIS na página(padrão:stretch)
  • Align-content = AGRUPA os itens e posiciona conforme a preferência (bloco de conteúdos)
  • Flex-grow = inserido na tag filho para crescer os itens de forma automática
    • 1 na especificidade CSS para todas as divs = crecscimento uniforme, mesmo tamanho
    • É possível deixar itens maiores mudando seus valores individualmente
  • Flex = especificar comportamento de cada tag div filho
    • É interessante colocar no *{} que o box-sizing será border-box, pois assim incorpora em cada caixa
